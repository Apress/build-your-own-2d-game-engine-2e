Multi-Texturing Support (based on Example 5.4)

 

To run:

·         WASD: move hero

·         Arrow and Shift+Arrow:

§   To move/scale the effect texture

 

Examine source code files:

1.       GLSL Shader:

a.       texture_vs.glsl: support one more vec2 attribute (the coordinate of the effect texture)

b.       texture_fs.glsl: support two texture sampler

                                                               i.      Second texture, when alpha is larger than 0.8 (sufficiently opaque), turn the pixel to semi-transparent

2.       engine/resources/texture.js (the module), support activating different texture unit

3.       engine/shaders /texture_shader.js: support the second texture sampler in the GLSL shader

a.       Note: the effect texture placement is accomplished by the placeAtWithSize() function that actually transform and send the UV values to the GPY

b.       The comments above this function says, a much more efficient implementation would compute a proper transformation matrix, load the matrix into the vertex shader, and then, send UV coordinate in the range of 0-1 to the attribute variable defined in the vertex shader.

c.       Similar to our initial issue with square showing as rectangle, when the aspect ratio of the Renderable (W/H) is not 1.0, the UV coverage must also be adjusted. When this is not handled properly, the rotation of the effect texture will appeared distorted. This is the last 5% of the assignment.

4.       engine/renderables /texture_renderable.js: support drawing with the second texture

a.       Remember the Shader object is shared among all existing Renderables. This means, the shader cannot keep any state on behalf of the Renderable. E.g., notice the placement information (mSecondTexturePlacement[], and the activation of the second texture lookup) must be kept in the Renderable, and only loaded to the Shader right before drawing.
